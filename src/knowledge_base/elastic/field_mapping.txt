================================================================================
SPLUNK FIELD MAPPING & COMMON INFORMATION MODEL (CIM)
Knowledge Base for SIEM Query Translator
================================================================================

OVERVIEW
--------
The Splunk Common Information Model (CIM) standardises field names across
different data sources. When translating queries from/to Splunk, use CIM
field names for portability.

================================================================================
CIM DATA MODELS AND THEIR FIELDS
================================================================================

AUTHENTICATION DATA MODEL
--------------------------
Model name: Authentication
Index hint: index=auth OR index=wineventlog

Core fields:
  action          — success | failure | error
  app             — application being authenticated to
  dest            — destination host/IP of authentication attempt
  duration        — duration of authentication event in seconds
  reason          — reason for failure (e.g., "bad password", "account locked")
  session_id      — unique session identifier
  signature       — human-readable name of event type
  signature_id    — numeric/string event ID (e.g., EventCode 4625)
  src             — source host/IP of authentication attempt
  src_ip          — source IP (explicit)
  dest_ip         — destination IP (explicit)
  tag             — CIM tag, always "authentication"
  user            — username attempting authentication
  user_agent      — user agent string (for web authentication)
  user_bunit      — business unit of the user

Common sourcetypes: WinEventLog:Security, linux_secure, access_log, radius

Windows EventCodes for Authentication:
  4624 — Successful logon
  4625 — Failed logon
  4634 — Account logoff
  4648 — Logon using explicit credentials
  4672 — Special privileges assigned to new logon
  4768 — Kerberos TGT requested
  4769 — Kerberos service ticket requested
  4771 — Kerberos pre-authentication failed
  4776 — NTLM authentication attempt

NETWORK TRAFFIC DATA MODEL
---------------------------
Model name: Network_Traffic
Index hint: index=network OR index=firewall

Core fields:
  action          — allowed | blocked | dropped | reset | teardown
  app             — application name (e.g., "http", "ssl", "dns")
  bytes           — total bytes transferred
  bytes_in        — bytes received
  bytes_out       — bytes sent
  dest            — destination host
  dest_ip         — destination IP address
  dest_port       — destination port number
  dest_zone       — destination network zone
  direction       — inbound | outbound | unknown
  duration        — connection duration in seconds
  dvc             — device (firewall/router) reporting the event
  dvc_ip          — device IP address
  flow_id         — unique flow identifier
  packets         — total packets
  packets_in      — packets received
  packets_out     — packets sent
  protocol        — network protocol (tcp, udp, icmp)
  src             — source host
  src_ip          — source IP address
  src_port        — source port number
  src_zone        — source network zone
  transport       — transport protocol
  ttl             — time to live
  vlan            — VLAN identifier

Web/HTTP specific fields:
  http_content_type   — MIME type of response
  http_method         — GET | POST | PUT | DELETE | HEAD | OPTIONS
  http_referrer       — HTTP referrer header
  http_user_agent     — browser/client user agent string
  response_time       — server response time in milliseconds
  site                — website domain
  status              — HTTP status code (200, 404, 500, etc.)
  uri_path            — path component of URL
  uri_query           — query string component of URL
  url                 — full URL
  url_domain          — domain from URL
  url_length          — character length of URL

DNS fields:
  answer              — DNS response value
  message_type        — query | response
  query               — queried domain name
  query_count         — number of queries
  query_type          — A | AAAA | MX | CNAME | TXT | PTR | NS
  record_type         — DNS record type in response
  reply_code          — DNS reply code (NOERROR, NXDOMAIN, etc.)
  reply_code_id       — numeric reply code

ENDPOINT / MALWARE DATA MODEL
-------------------------------
Model name: Endpoint
Index hint: index=endpoint OR index=sysmon

Process fields:
  action              — allowed | blocked
  command_line        — full command line of the process
  dest               — host where process ran
  file_hash           — hash of the executable (MD5, SHA1, SHA256)
  file_name           — executable filename
  file_path           — full path of executable
  integrity_level     — process integrity level
  parent_process      — parent process name
  parent_process_exec — parent process executable
  parent_process_id   — parent process ID (PPID)
  parent_process_path — parent process full path
  process             — process name
  process_exec        — process executable name
  process_guid        — globally unique process identifier
  process_hash        — hash of process binary
  process_id          — process ID (PID)
  process_name        — process name (short)
  process_path        — full path of process
  user                — user running the process

File system fields:
  file_access_time    — last access timestamp
  file_create_time    — file creation timestamp
  file_hash           — file hash (MD5/SHA1/SHA256)
  file_modify_time    — last modification timestamp
  file_name           — filename without path
  file_path           — full file path
  file_size           — size in bytes

Registry fields:
  registry_hive       — HKLM, HKCU, HKCR, HKU, HKCC
  registry_key_name   — registry key path
  registry_path       — full registry path
  registry_value_data — data stored in registry value
  registry_value_name — name of registry value
  registry_value_type — REG_SZ, REG_DWORD, REG_BINARY, etc.

INTRUSION DETECTION DATA MODEL
--------------------------------
Model name: IDS_Attacks
Index hint: index=ids OR index=snort OR index=suricata

Core fields:
  action              — allowed | blocked
  category            — attack category (e.g., "SQL Injection", "Malware")
  dest                — attack destination
  dest_ip             — destination IP
  dest_port           — destination port
  dvc                 — detecting device
  ids_type            — network | host
  severity            — critical | high | medium | low | informational
  signature           — name of the attack signature
  signature_id        — signature identifier
  src                 — attack source
  src_ip              — source IP
  src_port            — source port
  tag                 — ids_attack
  transport           — tcp | udp | icmp
  user                — user involved

CHANGE ANALYSIS DATA MODEL
---------------------------
Model name: Change
Index hint: index=change OR index=endpoint

Core fields:
  action              — created | modified | deleted | renamed | allowed | blocked
  change_type         — endpoint | network | application | directory
  command             — command executed to make change
  dest                — system where change occurred
  dvc                 — device reporting change
  file_name           — name of changed file
  file_path           — path of changed file
  object              — object being changed
  object_attrs        — attributes of object
  object_category     — file | directory | process | registry | port | service
  object_id           — identifier of changed object
  result              — result of change attempt
  src                 — source of change
  status              — success | failure
  user                — user making the change

EMAIL DATA MODEL
-----------------
Model name: Email
Index hint: index=email OR index=exchange

Core fields:
  action              — accepted | delivered | rejected
  dest                — recipient address(es)
  file_hash           — hash of attachment
  file_name           — attachment filename
  message_id          — unique message ID
  recipient           — list of recipients
  reply_to            — reply-to address
  sender              — sender email address
  sender_alias        — display name of sender
  size                — message size in bytes
  src                 — sending server IP
  src_user            — sender username
  subject             — email subject line
  tag                 — email
  transport_dest_port — destination port (25, 465, 587)

SPLUNK MACRO AND ALIAS PATTERNS
================================================================================

Common search macros (defined in transforms.conf / macros.conf):
  `authentication_indexes`    → index=auth OR index=wineventlog
  `network_indexes`           → index=network OR index=firewall OR index=proxy
  `endpoint_indexes`          → index=endpoint OR index=sysmon
  `cim_authentication`        → tag=authentication
  `cim_network_traffic`       → tag=network AND tag=communicate
  `cim_endpoint`              → tag=process OR tag=filesystem
  `threat_intel`              → inputlookup threat_intel.csv

Common field aliases (defined in props.conf):
  src_ip ALIAS for: src, Source_IP, SourceIp, c-ip, sourceip
  dest_ip ALIAS for: dst, Destination_IP, DestIp, s-ip, destinationip
  user ALIAS for: username, user_name, User, account, UserID
  bytes ALIAS for: bytes_total, total_bytes, Size, TotalBytes
